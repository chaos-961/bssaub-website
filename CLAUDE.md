# BSS AUB Website

Rebuild of bssaub.com for the AUB Business Student Society. One goal drives every decision: get an AUB student to grab the membership card. Everything else (sponsors, story, animations) exists to make that decision feel obvious.

Repo: `https://github.com/chaos-961/bssaub-website.git` · Live: `https://chaos-961.github.io/bssaub-website/` (GitHub Pages via Actions; custom domain later).
Current version: **0.3.3**.

## Rules (standing, no exceptions)

1. **Git gate.** Never commit or push unless the user explicitly says commit or push in that session. "Looks good" is not authorization. Every push to `main` auto-deploys — treat push authorization as deploy authorization and say so.
2. **Version contract.** Bump `version.json` by +0.0.1 as part of each authorized push (base-10 carry: 0.0.9 → 0.1.0). Keep the three static badge fallbacks (index/account/404 footers) in sync with it. Commit message format: `v0.1.3: short description`.
3. **No invented data.** Discounts, Instagram links, and redemption steps are never guessed. Unknown means blocked and listed, never filled with plausible garbage. Nothing marked "?" ever ships.
4. **No hyphens or dashes in user-visible copy.** Em dashes become commas, colons, or periods; titles join with "·"; compounds get rephrased ("hub led by students", not "student-led hub"). Applies to text nodes, aria-labels, alt text, titles, and meta.
5. **File content edits via the Edit/Write tools only.** PowerShell 5.1 text operations mojibake UTF-8.
6. **Feel checks on real hardware are the user's.** Headless verification covers logic, geometry, and colors; 60fps judgment is theirs.
7. **Session protocol.** Read this file, state version + open items in a line or two, await instruction. A user request is its own authorization — the rules stop AI initiative, not user freedom. Keep the version line and Open items section here current as work ships.

## Stack and shape

Vanilla JS + Vite 6 multi-page (`index.html`, `account.html`, `404.html`) · GSAP + ScrollTrigger · Lenis smooth scroll · Matter.js physics · Fraunces + Instrument Sans (self-hosted variable woff2). No other runtime deps, zero third-party scripts.

- `src/main.js` — init order matters: scroll → oceanMesh → nav → heroCard → modal → perkField → journey → sponsorCta → footerVersion → `scroll.refresh()` → preloader last (it collects `[data-preload]` images the field creates). The preloader `onComplete` fires `heroCard.enter()`. (The generative starfield background lived in this order until v0.2.5; it was extracted to the `Background Constellation` template and removed.)
- `src/data/sponsors.js` — single source of truth for every sponsor.
- `src/modules/*` — one module per concern; `src/styles/*` — tokens.css first, then per-section files.
- `public/assets/` — brand/, card/, sponsors/<category>/<id>.webp, journey/.
- `src/assets/bg/mesh.svg` — the generated background mesh. It lives in `src/`, not `public/`, so Vite resolves, hashes, and base-rewrites the `url()` in site-bg.css; a `public/` file would have to hardcode `/bssaub-website/` and break on the custom-domain flip. Regenerate with `node scripts/generate-bg-mesh.mjs [#hex]` (seeded, so output is byte identical unless the base changes). Never hand edit the SVG.
- `src/lib/mesh.js` — **the single source of truth for the mesh**: geometry, the palette ramp, the drift cycle, and the contrast maths. Imported by both `scripts/generate-bg-mesh.mjs` (writes the shipped SVG) and `src/modules/oceanMesh.js` (builds the WebGL buffers). Never reimplement or approximate it in a second place: the static SVG is the animation's **rest state**, and if the two drifted the background would visibly jump as the canvas fades in. `DEFAULT_BASE` must stay equal to `PALETTE_CYCLE[0]` for the same reason; the build script hard fails if they differ. **Recoloring the background is one edit**: those values, then rerun the script.
  - The AA gate lives in the build script and **refuses to write** on failure (exits 1, leaves the asset untouched), so an illegible color cannot reach the repo by accident. It checks both the still *and* the worst instant of the animated cycle with the shader dither applied. `--ink-soft` body copy on the darkest facet is the binding number, floor 4.5:1. A dev-only `COLORS` picker panel existed briefly at v0.3.2 and was removed the same day on the user's word; do not rebuild it without a fresh brief.
- Deploy: `.github/workflows/static.yml` (push to main → build → Pages). `vite.config.js` has `base: '/bssaub-website/'`. Custom-domain flip: CNAME in public/, base to `/`, revisit 404.html's absolute icon paths.
- Clean URLs (v0.1.3): internal links are extensionless (`account`, never `account.html`); Pages resolves them natively, and the `cleanUrls` middleware in `vite.config.js` (CLEAN_PAGES list) mirrors that in dev and preview. New pages join CLEAN_PAGES. Canonical and og:url use the extensionless form.
- Local run: `run-local.bat` (untracked, gitignored — user helper): opens the already-running server if 5173 is busy, else installs deps if needed and starts `vite --open`.

- Tokens in `src/styles/tokens.css` (`color-scheme: light`): `--bg #ffffff` (white paper; ran dark gray `#232225` v0.2.5 to v0.2.9 and `#2c2a2e` from v0.2.1, plain `#ffffff` v0.1.9 to v0.2.0, a warm `#fcfaf8` under the aurora v0.1.0 to v0.1.8) · `--surface #f8f3f5` (raised panels) · `--surface-glass rgba(255,255,255,0.78)` (nav token, still unused since the nav carries its own paper) · `--ink #221219` (~17:1 on bg) · `--ink-soft #5f4a54` (8.1:1, measured live) · `--line rgba(34,18,25,0.16)` · `--accent #881532` (the deep brand maroon sampled from `assets/logo.png`, 9.54:1 with white button text, measured live) · `--accent-strong #a31a3c` · `--accent-2 #b04a67` (5.22:1 on white). At v0.3.0 the lifted-for-dark glow rgbs `207,63,95` and `231,157,172` were swept back to the brand pair `136,21,50` and `176,74,103` across journey draws, modal, perk sparks, and button shadows; the dark ink shadows `34,18,25` were kept untouched through both flips (they were always depth pools, correct on either ground). The accent now matches the maroon brand assets instead of diverging from them, so `#cf3f5f` is retired entirely.
- **Background (v0.3.1, user brief): a low poly triangle mesh**, `div.site-bg` first in `<body>` on all three pages, styled by `src/styles/site-bg.css`, imported by all three entries (main.js, account.js, page.js). One fixed viewport-sized layer at `z-index -1`; `body` is `background: transparent` so it shows through, and `html` keeps the flat `--bg` for the overscroll gutter and as the load-failure fallback. Fixed rather than a document-tall background because stretching the mesh down a 10x-viewport page smears the facets into ribbons, and a fixed layer that never moves is the cheapest thing the compositor can hold. The **grain moved here from `body`** and rides as a second background layer at `background-blend-mode: multiply`; its noise tile is pre-compressed into `[0.94, 1.0]` by an `feComponentTransfer` in the data URI, because the old "dilute with a flat `--bg` layer on top" trick only works over a flat ground and would have washed the mesh 86% toward white.
  - **Legibility is the binding constraint, not taste.** Body copy (`--ink-soft`) sits directly on this on every page, so the palette is capped by the *darkest rendered facet*, which is not the darkest gradient stop: per-facet scatter pushes triangles below their sampled color. The first cut (deep stop `#e2bfcd`, scatter 0.055) measured `rgb(217,183,196)` and put `--ink-soft` at **4.44:1**, under the AA floor. Shipped values measure `rgb(224,194,206)`: `--ink-soft` 4.93:1, `--ink` 10.93:1, `--accent` 5.81:1, `--accent-2` focus ring 3.18:1. **Re-run the contrast check against the rendered SVG if you retune; the stop values alone do not tell you.**
  - Square 1400x1400 source with `background-size: cover` so it crops gently in both orientations (a landscape source would double the facet size on portrait phones). Facets land ~41px on a 375px phone and ~64 to 128px on desktop, i.e. 9 to 20 across, bracketing the ~12 of the brief's reference. `height: 100lvh` so a retracting mobile URL bar cannot expose a strip. ~8.5KB gz, vector so one file is exact at every DPR.
  - Cost of the static layer: no JS, no animation, no `mix-blend-mode`, no `backdrop-filter`. Painted once and re-used as content scrolls over it.
- **The ocean layer (v0.3.2, user brief: the mesh should swell like water and the color should drift).** `src/modules/oceanMesh.js` adds `canvas.site-bg__canvas` inside `div.site-bg` and animates the same lattice in WebGL. The static SVG **stays underneath as the floor** and is what paints pre-bundle, with JS off, under `prefers-reduced-motion`, when WebGL is missing, and on context loss; the canvas only gets `is-live` (opacity 1) after frame zero is drawn, so there is no flash and the crossfade is between two states of one mesh, not two designs.
  - **WebGL is the cheap option here, not the expensive one.** Canvas 2D would mean 800 `fill()` calls per frame on the main thread, competing with Lenis, GSAP and the Matter.js field; SVG/CSS cannot move vertices at all. WebGL is one draw call of 2400 vertices with displacement in the vertex shader. Measured: draw submission ~0.0003ms, and the fragment shader is one varying plus a hash.
  - What holds the cost down, all of it load bearing: **one draw call**, buffers uploaded once (only uniforms change per frame), **30fps cap**, backing store pinned to **DPR 1** (adjacent facets differ by a few levels so there is nothing for extra resolution to sharpen, and it is a 4x fragment saving on 2x displays), fully parked on `visibilitychange`, and torn down on context loss. **Do not add a per frame `getBoundingClientRect`** — sizing is driven by a debounced `ResizeObserver`, because reading layout every frame would make the background the most expensive thing on the page.
  - Sizing uses `ResizeObserver` on the canvas, not a window `resize` listener: this fixed layer changes width by the scrollbar once the perk field grows the page past one screen, which no window event reports.
  - **Not weird** is a verified property, not a hope. Amplitude is 7 mesh units against a 70 unit cell; simulated over a 60s sweep across all 800 facets, peak vertex displacement is 15.8 units (22.6% of a cell), **zero triangles ever invert**, and the smallest facet still holds 0.81x its rest area. The mesh is drawn at `OVERSCAN` 1.04 (28 units of margin) so displaced edge vertices can never pull a gap in at the screen edge. Push amplitude much past this and facets start crossing, which reads as static noise rather than water.
  - Color drift walks `PALETTE_CYCLE` (three stops, near equal luminance so the page breathes hue rather than brightness) over ~96s, plus a ±0.06 slide of the ramp itself so light appears to move across the surface. Both are bounded so the deep end still clamps to the same stop and the contrast budget holds at every instant.
- Background was a generative starfield (v0.2.3 to v0.2.4), extracted at v0.2.5 into the `Background Constellation` template (see the theme paragraph above) and removed from the site. The template carries the final, improved version (pen 40% faster, `excludeSelectors` so lines never draw behind an image). If a starfield is ever wanted here again, pull it from `C:\Development\Website Templates\Background Constellation`, do not rebuild it.
- The navbar is a "paper" sheet (v0.2.2, user call) carrying the same grain as the page via `mix-blend-mode: multiply` at opacity 0.14, with dark ink content drawn from nav-scoped light tokens (`--nav-paper`, `--nav-ink #221219`, `--nav-ink-soft`, `--nav-accent #881532`, `--nav-line`) set on `.site-nav, .nav-menu` so both the bar and its dropdown flip together. Those scoped tokens duplicate the global ones almost exactly now that the page ground is light, and they **stay** on purpose: declaring the bar's palette independently is what carried it through the last two ground flips without a rewrite. **Separation (v0.3.1, user report that the bar read as invisible):** when the ground went white at v0.3.0 the bar's white fill and the page composited to the same value (~250/255), leaving only a 14% hairline (a 1.28:1 edge) to mark it. Three things carry it now, none heavy alone: `--nav-paper` moved off pure white to the warm raised-panel tone `#f8f3f5`, the hairline went to 24% (`rgb(211,208,209)`, 1.53:1), and an always-on two-layer lift shadow was added; `.is-scrolled` deepens the last two (30%, `rgb(188,183,186)`, 1.98:1). `.is-hidden` widened from 16px to 24px so the deeper shadow still cannot peek at the top edge. The brand maroon logo reads on the paper with no filter; the v0.2.1 `brightness(1.5)` lift was for the old dark glass nav and is gone. Scrolling past 80px lifts the bar with a shadow (`.is-scrolled`) and no longer swaps the fill; on top of that the bar hides on scroll down and reveals on scroll up again since v0.2.4 (`.is-hidden` translates it up past its shadow, user call reinstated the v0.1.4-removed behavior; stays put near the top and while the menu is open, `focusin` brings it back for keyboard users). Sponsor bubble tiles are opaque coins whose marks contrast with their own baked field, so they read as is on either ground and needed no work in the v0.3.0 flip. The preloader keeps the maroon brand curtain (`#290a13`, inline critical CSS in index.html) that wipes up to reveal the white page. OG image and favicons stay maroon brand assets, and the accent now matches them again.
- Type: Fraunces (display, `opsz 144`, WONK on hero/stub titles) + Instrument Sans (UI/body). Fluid clamp() scale lives in tokens.
- **Section rhythm (v0.3.2, user report of a big gap between sections): `--section-pad` was halved**, `clamp(4.5rem, 10vw, 8.5rem)` to `clamp(2.25rem, 5vw, 4.25rem)` (136px to 68px on a 1440px desktop). One token drives every boundary (hero bottom, perk field, journey, sponsor CTA, agency credit), so that is the only knob. Measured at 1440x900: divider to "THE PERK FIELD" 137px to 69px, trust row to that eyebrow 286px to 184px. It reads as 36% and not 50% on that second number because the hero is `min-height: 100svh` with `align-items: center`, so shrinking its bottom padding by X only lowers the space under the facts row by X/2, the rest redistributes above. The residual ~81px under the hero content is centering slack, not padding, and no `--section-pad` value can reach it.
- Motion: power2/power3-out reveals; elastic feel only inside the Perk Field; scrub only where scroll is the mechanic (field impulses, journey path); the background ocean swells on its own clock, unrelated to scroll. `prefers-reduced-motion` strips physics, tilt, scroll effects, and the background canvas sitewide; `?reduced-motion` URL param is the QA hook (verified at v0.3.2: no canvas is built and the static mesh with its grain remains).
- **GPU floor (v0.2.9, user report of ~30% GPU at idle).** Two rules hold the line, and both are easy to break by accident:
  1. **No blend or backdrop layer that spans the viewport.** `mix-blend-mode` and `backdrop-filter` read their backdrop, so the compositor snapshots and reprocesses everything behind them on every frame anything moves. The grain uses `background-blend-mode` (above), the nav burger's decorative `backdrop-filter` over an already opaque bar is gone, and the modal's full-screen blur is scoped to `.is-open` so no backdrop root stands by while it is closed. The nav's own `::before` grain multiply is fine: it is 72px tall and blends against its own fill. The v0.3.1 mesh layer is viewport-sized but does **not** break this rule: it blends its own two background layers inside its own paint and never reads what is behind it, which is exactly why the grain there is `background-blend-mode` and not a `mix-blend-mode` sheet.
  2. **No `filter` on a scrubbed or looping element, and no infinite animation off screen.** The journey's draw path carried `blur(5px)` plus a `drop-shadow`, whose filter regions were the path bbox (the whole ~1080x2000px track) re-rendered on every scroll frame; it is now three plain stacked strokes (`--halo` / `--glow` / `--draw`, all scrubbed together from `litPaths` in journey.js) that fake the falloff in the rasterizer. Their opacities were lifted at v0.3.0 (`0.06` to `0.10`, `0.14` to `0.20`) because on the white ground the same stroke subtracts light instead of adding it, and 6% simply vanished; that restores the dark ground's bloom, not more. The ambient ember particles (14 desktop / 8 mobile drifting dots in `.journey__particles`) lost a `blur(0.5px)` here at v0.2.9 and were **deleted outright at v0.3.3** on the user's word (they read as bubbles popping in and out around the Join checkpoint); the wrapper div, the `.journey__particle*` CSS, and the `drifts` array in journey.js are all gone. Everything that still loops forever is gated to on screen: hero card idle float (IntersectionObserver in heroCard.js), orb halo pulse (`.journey__marker.is-active`), and the Join CTA glow (`.journey.is-onscreen`, toggled by journey.js) which animates `box-shadow` and so repaints, not just recomposites.
  Perk field per step: the active-item set is built once into a reused array, `isMobile()` resolves once a frame, and catch-up is capped at 2 substeps so a slow frame cannot stampede the next one.
  3. **The v0.3.2 ocean is the one sanctioned always-on animation, and it earns it.** It is the background, so "gate it to on screen" means gating it to the tab being visible, which it does via `visibilitychange`. It stays within the floor by being one draw call at 30fps at DPR 1 with a trivial fragment shader, and by never reading layout in its loop. If GPU cost is ever reported again, this is the first thing to measure but not automatically the culprit: check for a new viewport-spanning blend or a `filter` on a scrubbed element first, since those were the actual causes at v0.2.9.

## Sponsor data

One object in `src/data/sponsors.js` + one image = a new sponsor. Fields: `id`, `name`, `category` (restaurants | clothing | health-beauty | fitness | services), `discount` (string, or null to render badge-less with no % in the aria-label), `instagram`, `image`, `size` (relative bubble scale), and optional `details { summary, notes, steps, links }` — the presence of `details` is what turns a bubble into a popup instead of a direct link. Image filename equals the id: `assets/sponsors/<category>/<id>.webp`, about 320px square, ≤25KB.

## Budgets and floors

- LCP under 2.5s on throttled 4G (live median 1.91s measured at v0.0.2) · CLS ~0 (all media has reserved dimensions).
- JS ≤ 100KB gz — **~94KB at v0.3.3 and no longer roomy**: the ocean layer added ~3.2KB gz (module plus the shared mesh geometry). Anything substantial from here needs a lazy chunk or something else has to leave. · card art ≤ 120KB · bubbles ≤ 25KB each · background mesh ≤ 15KB gz (8.5KB; one vector file serves every DPR, so this is the whole background budget).
- Accessibility: semantic landmarks, skip link, focus traps, visible focus everywhere, every bubble labeled ("Name, 20% OFF, opens Instagram"), token pairs hold AA+ contrast on the ground (measured live at v0.3.0 on flat white: body ink 17:1, `--ink-soft` 8.1:1, white on `--accent` 9.54:1, `--accent-2` 5.22:1). Any change to the mesh palette or to `--ink-soft` has to be re-measured there.
